MLB Stats CLI — How to Use
===========================

A local command-line tool that looks up an MLB player or team, pulls
their game-by-game stat log or schedule for a season, and plots a
chosen stat over time with a rolling average — for one player/team, or
two compared side by side.


SETUP
-----

1. From the project root, activate the virtual environment:

     cd /home/ziz/development/PythonBaseball
     source venv/bin/activate

2. If you haven't installed the package yet (or made changes to the
   source files and need it re-linked), run:

     pip install -e .

   The -e flag installs it in "editable" mode, so any edits to the
   files in mlb_stats/ take effect immediately without reinstalling.


BASIC USAGE
-----------

     mlb-stats "Shohei Ohtani"

This looks up the player by name, pulls their game log for the
default stat (era) and default season (2026), and opens a chart
window showing:
  - the season's cumulative value of the stat (gray line)
  - a rolling 5-game average (red line)


SUPPORTED STATS (--stat)
-------------------------

Pitching:
  era    Earned Run Average
  whip   Walks + Hits per Inning Pitched
  k9     Strikeouts per 9 innings
  bb9    Walks per 9 innings
  fip    Fielding Independent Pitching

Batting:
  avg    Batting average
  obp    On-base percentage
  slg    Slugging percentage
  ops    On-base Plus Slugging (obp + slg)

Team:
  win_pct  Win percentage over the season

     mlb-stats "Paul Skenes" --stat k9
     mlb-stats "Freddie Freeman" --stat obp
     mlb-stats "Los Angeles Dodgers" --stat win_pct

The tool automatically requests the right stat group (pitching,
batting, or team) from the API based on which --stat you pick -- and
for a team stat, the player/player2 arguments mean team names instead
(matched the same partial/case-insensitive way, e.g. "dodgers" or "LA
Dodgers" both work).

A note on FIP: unlike most other stats here, the MLB Stats API doesn't
return a season-cumulative FIP value, so the "season cumulative" line
is computed locally from cumulative HR/BB/HBP/K/IP. Also, FIP's
constant term (which puts it on the same scale as ERA) is normally
recalculated by MLB every season from league-wide totals -- this tool
uses a fixed, commonly-cited approximation (3.10) rather than the
exact value for whichever season you're looking at, so FIP numbers
will be close but not perfectly official.

A note on win_pct: also computed locally rather than from the API's
own (pre-rounded) season record, for full precision.


COMPARING TWO PLAYERS (OR TEAMS)
----------------------------------

Pass a second player (or team, for --stat win_pct) name to overlay
both rolling averages on one chart instead of plotting a single one:

     mlb-stats "Shohei Ohtani" "Paul Skenes" --stat era
     mlb-stats "Los Angeles Dodgers" "San Francisco Giants" --stat win_pct

Both must have data for the same --stat (and therefore the same stat
group) and --season, or you'll get a "No ... data found" error for
whichever one doesn't. By default the comparison chart shows only the
rolling lines (no cumulative lines) to keep it readable — see
"COMPARISON DISPLAY OPTIONS" below to change this.


COMPARISON DISPLAY OPTIONS
----------------------------

These only apply when a second player is given.

--layout {overlay,stacked,side-by-side}
     How to arrange the two players. Default: overlay.

       overlay       Both players on one shared axes (the default —
                      matches the original comparison view).
       stacked       One axes per player, stacked vertically. Each
                      player gets their own y-scale, so this is best
                      for seeing each player's own hot/cold streaks
                      clearly, at the cost of not being directly
                      scale-comparable.
       side-by-side  One axes per player, side by side, sharing a
                      y-axis — best for directly comparing scale.

       mlb-stats "Shohei Ohtani" "Paul Skenes" --stat era --layout stacked
       mlb-stats "Shohei Ohtani" "Paul Skenes" --stat era --layout side-by-side

--show-cumulative
     Also draw each player's season-cumulative line (dashed, lower
     alpha) alongside their rolling line. The cumulative line hides
     the same early-season warm-up period the rolling line already
     omits (fewer than --window games played), since a rate stat from
     a handful of games can be extremely noisy and would otherwise
     dwarf the rest of the season on the shared axis.

       mlb-stats "Shohei Ohtani" "Paul Skenes" --stat era --show-cumulative

--diff
     Add a panel below the main chart showing player 1's rolling
     value minus player 2's, shaded by whoever is ahead at each point.
     Since the two players are rarely on the same game schedule, this
     reindexes both onto the union of both players' game dates and
     forward-fills each player's rolling value between their games.

       mlb-stats "Shohei Ohtani" "Paul Skenes" --stat era --diff

All three combine freely:

       mlb-stats "Shohei Ohtani" "Paul Skenes" --stat era \
           --layout stacked --show-cumulative --diff


OPTIONS
-------

--stat NAME
     Which stat to plot. See "SUPPORTED STATS" above. Default: era.

--season YEAR
     Choose a different season.

       mlb-stats "Clayton Kershaw" --season 2020

--window N
     Change the rolling average window (number of games averaged
     together). Smaller windows show every hot/cold streak but look
     noisy; larger windows smooth the trend but lag behind recent form.

       mlb-stats "Shohei Ohtani" --window 3
       mlb-stats "Shohei Ohtani" --window 8

--save [FILE]
     Save the chart to a PNG file instead of opening a display window.
     Useful for headless machines or just keeping a permanent copy.
     Works for both single-player and comparison charts.

       mlb-stats "Clayton Kershaw" --save kershaw.png
       mlb-stats "Shohei Ohtani" "Paul Skenes" --stat era --save compare.png

     Leave FILE off to auto-generate a name from the player(s), stat,
     and season, e.g. shohei-ohtani_era_2026_2026-07-07.png or
     shohei-ohtani_vs_paul-skenes_era_2026_2026-07-07.png. --window,
     --layout, --show-cumulative, and --diff get appended to the name
     only when you've changed them from their defaults, e.g.
     shohei-ohtani_vs_paul-skenes_era_2026_w10_stacked_cumulative_diff_2026-07-07.png.
     The trailing date is today's date (when the plot was generated,
     not the season), so re-running the same command tomorrow won't
     overwrite today's file. Saves into the current directory and
     overwrites an existing file of the same name, same as passing
     that name explicitly would.

       mlb-stats "Shohei Ohtani" --save
       mlb-stats "Shohei Ohtani" "Paul Skenes" --stat era --layout stacked --diff --save

--table
     Also print the per-game data behind the chart as a plain text
     table (date, opponent, that game's own value, season-cumulative
     value, rolling value) before opening/saving the chart. In
     comparison mode, prints one table per player, each labeled with
     the player's name.

       mlb-stats "Shohei Ohtani" --table
       mlb-stats "Shohei Ohtani" "Paul Skenes" --stat era --table

     Rows before enough games have been played to fill the rolling
     window show NaN in the rolling column — this matches the small
     gap you'll see at the start of the rolling line on the chart.


PITCH VELOCITIES (--velo)
---------------------------

Instead of a stat over time, plot every single pitch a pitcher threw:
one column of dots per game (colored by pitch type), with a dashed
line tracing each game's max velocity across the stretch -- so both a
pitcher's velocity distribution and their max velo trend are visible
at a glance:

     mlb-stats "Paul Skenes" --velo
     mlb-stats "Paul Skenes" --velo --start-date 2026-06-01 --end-date 2026-06-30
     mlb-stats "Paul Skenes" --velo --table --save

The player must be a pitcher. --start-date/--end-date (inclusive,
YYYY-MM-DD, either side optional) narrow the plot to a stretch of the
season -- e.g. one month, as above. --season and --save work as usual;
--table prints a per-game summary (pitch count, max/median/min velo)
rather than one row per pitch. --stat, --window, and the comparison
options don't apply, and a second player isn't supported.

Unlike the game-log stats, this pulls each game's full play-by-play
feed (one extra API call per game, cached for 24 hours -- a completed
game's pitches never change). The velocity is the pitch's release
speed ("startSpeed") as tracked by MLB's Statcast system; on the rare
pitch the tracking missed, that pitch is simply omitted.


TEAM STANDINGS (--standings)
------------------------------

Instead of a per-game time series for a single player/team, show a
whole division's current standings as a horizontal bar chart (win
percentage, longest bar on top) and/or a table:

     mlb-stats --standings "AL East"
     mlb-stats --standings "National League West" --table
     mlb-stats --standings "NL Central" --season 2024 --save

No player argument is needed (or used) in this mode. "AL"/"NL" are
expanded to "American League"/"National League" automatically, and
matching is partial/case-insensitive like team names elsewhere, so
"al east", "AL East", and "American League East" all work. --table,
--save, and --season all behave the same as in single-player mode;
--window, --layout, --show-cumulative, and --diff don't apply here
since there's no rolling time series.

       mlb-stats --standings "AL East" --table --save east.png


COMBINING OPTIONS
-----------------

     mlb-stats "Shohei Ohtani" --stat whip --season 2025 --window 3 --save ohtani_whip.png
     mlb-stats "Shohei Ohtani" "Freddie Freeman" --stat avg --window 10
     mlb-stats "Shohei Ohtani" "Paul Skenes" --stat era --table --save compare.png
     mlb-stats "Shohei Ohtani" "Paul Skenes" --stat era --layout side-by-side --show-cumulative --diff


TROUBLESHOOTING
----------------

"No player found for '...'"
     Check the spelling of the name. The lookup uses MLB's player
     search API, so it needs to reasonably match a full name.

"No team found for '...'"
     Same idea, for --stat win_pct: check the spelling. Partial
     matches work (e.g. "dodgers", "LA"), but very short/ambiguous
     queries may match the wrong team -- see "Multiple matches" below.

"No {pitching,batting} data found for player ID ... in ..."
     The player has no game log for that stat's group and season
     (e.g. a position player for a pitching stat, or a season where
     they were injured/didn't appear/hadn't debuted). Try a different
     --stat or --season.

"No completed games found for team ID ..."
     The team hasn't played any games yet this season (or the season
     hasn't started) -- try a different --season.

"No division found for '...'"
     Check the spelling, e.g. "AL East", "NL Central". See "TEAM
     STANDINGS" above for the alias rules.

"No pitch data found for player ID ..."
     --velo: the player has games in the range but no tracked pitches
     -- most likely they're a position player, not a pitcher.

"No games found between ... and ..."
     --velo: no games in the --start-date/--end-date range. Check the
     dates are YYYY-MM-DD and fall within the season.

"Multiple matches, using: ..."
     Just a heads up — the name (or team) search found more than one
     match and picked the first one. Use a more specific name if it
     picked the wrong one.

"FigureCanvasAgg is non-interactive, and thus cannot be shown"
     Harmless if you used --save. If you want an actual display
     window, make sure python3-tk is installed on your system
     (sudo apt install python3-tk on Debian/Ubuntu) so matplotlib can
     pick a GUI backend.


PROJECT LAYOUT
---------------

mlb_stats/stats.py   Registry of supported stats (STAT_CONFIGS). Each
                      entry declares its stat group, its season-
                      cumulative field name, and the per-game fields
                      to sum for a correct rolling calculation.
                      Add a new stat here — nothing else needs to
                      change to support it.
mlb_stats/api.py      API calls: find_player(), get_game_log(),
                      find_team(), get_team_schedule(), find_division(),
                      get_division_standings(), get_game_pitches()
mlb_stats/plots.py    Data shaping + charting: build_stat_dataframe(),
                      build_team_win_dataframe() (flattens a team
                      schedule into the same shape build_stat_dataframe
                      produces, so everything downstream is unaware
                      whether it's looking at a player or a team),
                      add_rolling_stat(), format_stat_table(),
                      plot_stat(), plot_stat_comparison()
                      (COMPARISON_LAYOUTS lists the valid --layout
                      values for comparison mode), the standings
                      trio: build_standings_dataframe(),
                      format_standings_table(), plot_standings(),
                      and the pitch-velocity set:
                      filter_splits_by_date(), build_pitch_dataframe(),
                      format_pitch_table(), plot_pitch_velocities()
mlb_stats/cli.py      Argument parsing / entry point (main())

Data source: MLB Stats API (https://statsapi.mlb.com) — public,
unofficial, no API key required.
